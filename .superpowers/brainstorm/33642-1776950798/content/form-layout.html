Admission Form Layout

The current form is a 3-step modal dialog (max-height 60vh). With 30+ fields, it's cramped. Which approach feels right?

A

Full Page (Dedicated Route)

A standalone page at /students/admit with a stepper/wizard layout. Full screen width, no scrolling constraints. Back button returns to student list. Best for complex, data-heavy forms.

Pros

  • Maximum space for fields
  • No scroll constraints
  • Can show progress sidebar
  • Feels like a proper workflow

Cons

  • Navigates away from student list
  • Slightly more navigation overhead
B

Side Panel (Slide-over Sheet)

A wide sheet sliding in from the right (~60-70% width). Student list stays visible behind. Steps shown as vertical tabs on the left side of the panel.

Pros

  • Context preserved (list visible)
  • More space than modal
  • Modern SaaS feel

Cons

  • Still constrained width on smaller screens
  • Can feel cluttered with many fields
C

Improved Modal (Keep Current, Enhance)

Keep the dialog but make it larger (max-width ~900px, taller), improve field grouping with collapsible sections, and add a visual stepper bar at the top.

Pros

  • Minimal code changes
  • Familiar pattern
  • Quick to implement

Cons

  • Still fundamentally constrained
  • Many fields in limited space
